# Deferred Lowerer Tasks

This file catalogs every task that was deferred during Slices 0–10 of the
lowerer migration. **All "Critical" items must be resolved before the
`InkParser` fallback in `CompilationAnnotator` can be deleted.** Items in
later sections are correctness gaps that should be triaged — some may be
acceptable permanent limitations, others must land before parity with the old
parser.

Each item lists the originating slice in `[ ]` so it can be cross-referenced
against the conversation history. Resolved items are marked ✅ with a brief
note on how they were addressed.

---

## Critical (block InkParser removal)

These node types currently fall through to InkParser. Deleting the fallback
without lowerers for them will silently drop content.

- ✅ **`BlockDialogue` / `BlockHeading` / `BlockTitle` / `BlockTransitional` / `BlockWrite` / `BlockAction`** [Slice 4] — block lowerers added to `lowerDisplay.ts` mirroring the inline forms. Body extracted by `extractBlockBody` which strips per-line indentation; the existing `buildDisplayContent` then runs the text through `processDisplayText` for escape/break/newline handling.
- ✅ **`ImageLine` / `AudioLine` / `ImageAndAudioLine` (asset lines)** [Slice 4] — new `lowerAssetLine.ts`. Each `[[...]]` / `((...))` command emits a `Tag(start)` + `Text "image:..."` (or `"audio:..."`) + `Tag(end)`, followed by a trailing `Text "\n"`.
- ✅ **`LuauEndKeyword`** [Slice 10] — explicit `case "LuauEndKeyword": return {};` in the dispatcher. Future enhancement: use it as an explicit scope-termination signal in `parseIncrementally` to make `end` close a flow scope rather than relying on the next declaration to implicitly do so.
- **`LuauForLoop` / `LuauWhileLoop` / `LuauRepeatLoop` / `LuauDoBlock`** [Slice 10] — currently stubbed to `{}`. Bodies are dropped. Real lowering needs a runtime design: ink has no native loops, so this requires desugaring to conditional + divert patterns (recursive knots), or extending the runtime with loop primitives.

## Expression-layer gaps

- ✅ **Interpolated strings (` `text {expr}` `)** [Slice 5] — `lowerInterpolatedString` in `lowerExpression.ts` walks the `LuauInterpolatedString_content` children, accumulating `Text` for literal segments and lowering each `LuauInterpolatedStringExpression` into the `StringExpression.content`. Each inner `Expression` is marked `outputWhenComplete = true` so the runtime emits its value into the open string slot between `BeginString`/`EndString`. Same machinery powers `{...}` interpolation inside display lines (see resolved entry in Display-layer gaps).
- ✅ **Tables (`{ key = value }` → `ObjectExpression`)** [Slice 5] — added `ObjectValue` (a real `Value<Map<string, AbstractValue>>`) to the runtime alongside a new `ValueType.Object`. Built `BeginObject`/`EndObject` control commands: `BeginObject` pushes a marker onto the eval stack; each entry pushes a `StringValue(key)` followed by the evaluated value; `EndObject` walks the stack back to the marker, assembles the `ObjectValue`, and pushes it. `ObjectExpression.GenerateIntoContainer` emits this sequence; nested tables work because each inner `EndObject` leaves a single `ObjectValue` on the stack that the outer iteration treats as a normal value. JSON serialization writes/reads `{"obj": {key: …}}`. The lowerer `lowerTable.ts` walks `LuauTable_content`, pairing each `LuauAccessPath` key with the following `LuauAssignmentOperation` value. Bracket-key form (`[expr] = value`) and pure array-style tables are still rough — keys are auto-incremented strings for bare expressions.
- ✅ **Property indexers (`obj[key]`)** [Slice 5] — added a runtime `IndexValue` control command that pops `key` + `base` and pushes `base[key]`, working over `ObjectValue` and (1-indexed) `StringValue`. New `IndexExpression(base, key)` ParsedHierarchy class emits `base; key; IndexValue`. In `lowerAccessPath`, any path containing a `LuauPropertyIndexer` switches to building an `IndexExpression` chain rooted at the leading dot-path; later dot-property segments after an indexer fold to string-key index ops (since the base is now a value, not a namespace).
- ✅ **Dot-access on table-typed values (`obj.field` as property read)** — `VariableReference.ResolveReferences` now accepts a multi-segment path when the first segment resolves as a variable (arg / temp / global) and isn't a constant or struct. The runtime side (`Story.ts > VariableReference` branch) walks `obj.field.nested` as a chained map-index, mirroring the existing `lang.current` pattern. Authors can now write `obj.field` against a stored table directly without the `obj["field"]` bracket-indexer detour. Validated by `smoke.test.ts > table-typed store: read + property-access via dotted name`.
- **Recursive discard-call hang in pure functions** — `& fn(args)` recursion inside a `function ... end` body causes `Story.ContinueInternal` to never return (the worker eventually crashes with stack overflow / heap exhaustion). Non-recursive discard-calls work, and recursive functions that use `return` value-form (like `function fact(n) if n<=1 then return 1 else return n * fact(n-1) end end`) work fine. The bug is specifically the discard-call recursion shape. Blocks `Evaluation.test.ts > factorial by reference` and `Logic.test.ts > nested pass by reference` — both upstream ink fixtures use the discard-call recursion shape with `ref` parameters, which the table-as-pseudo-ref rewrite preserves. Workarounds today: rewrite recursion as iteration, or use value-returning recursion if the function semantics permit.
- ⚠️ **Method calls (`obj:method(args)` via `LuauFunctionAccessor`)** [Slice 5] — _call-site lowering done; receiver-type dispatch deferred._ `collectTokens` peeks at adjacent siblings: when it sees a `LuauAccessPath` ending with `LuauFunctionAccessor` immediately followed by a sibling `LuauParenthetical`, `lowerMethodCall` combines them into `FunctionCall("method", [obj, ...args])` — the receiver becomes the implicit first parameter. This works today as a flat-namespace call.

  **Intended language design** — the real home for methods is a `define` block, which is sparkdown's shorthand for "object with methods" (and inheritance):

  ```sparkdown
  define Bird with
    canFly = true
    isFlying = false
    fly()
      self.isFlying = true
      print("I'm a bird and I'm flying!")
    end
  end

  define Penguin as Bird with
    canFly = false
    isSwimming = false
    fly()
      print("Penguins can't fly...")
    end
    swim()
      self.isSwimming = true
      print("I'm a penguin and I'm swimming!")
    end
  end
  ```

  In the intended model, `p:swim()` on a `Penguin` instance dispatches to `Penguin.swim` (or up the `as Bird` chain if not overridden), with `self` bound to `p`. The current call-site lowering only handles the desugar half — it does NOT do receiver-type dispatch. To fully realize the design, several blocked items have to land together:
  1. ✅ Call-site sugar (`obj:method(args)` → call with `obj` as first arg). _Done._
  2. **`LuauFunctionDefinition` inside `LuauDefine` captured as a method member** — see _Function members in defines_ under "Define-layer gaps".
  3. **Inheritance via `define X as Y with ...`** — `StructDefinition.type` already records the parent name; method lookup needs to walk that chain.
  4. **Runtime method dispatch on the receiver** — the call site needs to look up `<receiver-type>.<method-name>` rather than the flat global `<method-name>`. Probably implemented by emitting a `FunctionCall` whose name is computed from the receiver's struct type at runtime, or by adding a new "MethodCall" runtime construct.
  5. **`self` binding** — methods need their `self` parameter wired up automatically (the grammar's `swim()` form has no parameter list at the source level).

  Until 2–5 land, write methods as plain global functions whose first parameter is the receiver; the `:` call sugar already works on them. After 2–5 land, the global-function form remains a valid fallback for utilities that aren't naturally part of a type.

- ✅ **Chained call + access (`f(x).y`, `obj.method().more`)** [Slice 5] — `lowerAccessPath` now detects when a `LuauAccessPath` either contains a `[key]` indexer or pairs a function call with trailing access parts, and routes those cases through `lowerValueChainAccessPath`. That builder starts the chain from either a leading `FunctionCall` or a leading `VariableReference` and wraps each subsequent `LuauPropertyAccessor` / `LuauPropertyIndexer` in an `IndexExpression`. Pure dotted/identifier paths still lower to a single `VariableReference` so ink's hierarchical name resolution stays intact.
- ✅ **Nil literal (`LuauNil`)** [Slice 5] — added to `PRIMARY_NODES` and lowered to `NumberExpression(0, "int")`. Ink already treats `0`/empty as falsy, so `x == nil` round-trips as `x == 0` and `if x then` / `if not x` work without further changes. A dedicated `NilValue` runtime type was considered but deferred — there's no user-facing scenario yet that needs to distinguish nil from numeric zero.
- ✅ **TypeCast operator (`::`)** [Slice 5] — moved `LuauTypeCastOperation` (and `LuauTypeAnnotationOperation`) out of the binary-op wrappers and into a `TYPE_ONLY_WRAPPERS` set whose entire subtree is skipped during token collection. The `::` operator entry was dropped from the Pratt parser's precedence table. `a :: SomeType` now lowers to just the `a` expression, which is the correct runtime behavior for a type-only annotation.
- ✅ **Length operator (`#`)** [Slice 5] — registered a new `LEN` native function in `NativeFunctionCall`, with unary ops for `String` (`x.length`), `List` (`x.Count`), and `Object` (`x.size`). Added `ValueType.Object` dispatch to the `Call()` switch and `AddStringUnaryOp` / `AddObjectUnaryOp` helper methods. `UnaryExpression.nativeNameForOp` maps `#` → `LEN`. Fixtures cover `#items`, `#{a=1,b=2}`.

## Statement-layer gaps

- ✅ **Bare-expression statements inside function bodies** [Slice 10.5] — handled at the **lowerer level** rather than as a dedicated grammar rule. The grammar lets the LHS and RHS parse as sibling `LuauAccessPath` + `LuauAssignmentOperation` nodes; `lowerStatements` peeks at the next non-trivia sibling after an `LuauAccessPath` and, if it's a `LuauAssignmentOperation`, combines them via `lowerImplicitAssignmentStatement(path, op, ctx)`. This mirrors Lua's recursive-descent parser, which parses a "suffixed expression" first and then disambiguates based on whether `=` follows. The lowerer approach handles arbitrary access-path shapes — including the user's tricky case `this.is["a"].access.path = value` and bracket-indexer targets like `state.items["sword"].count += 1` — without trying to encode them in a regex lookahead. A grammar-based attempt was tried first (`LuauImplicitAssignmentStatement` rule with a lookahead) but rejected because regex can't easily express `[expr]` indexers with balanced delimiters. Simple-identifier LHS lowers to `VariableAssignment(kind: "reassign", ...)`; multi-segment LHS routes through `lowerPropertyTargetAssignment`. Compound assignment (`x += y`, `obj.f += y`) desugars to `x = x + y` / `obj.f = obj.f + y`.
- ✅ **`& foo()` expression-statement** [Slice 6] — `lowerExplicitStatement` now handles the no-assignment case: lower the `LuauAccessPath` via `lowerExpressionFromNodes`, and if the result is a `FunctionCall`, set `shouldPopReturnedValue = true` so the runtime evaluates the call for side effects and pops the unused return off the eval stack. Non-call bare expressions (`& foo`, `& 1 + 2`) are dropped — ink only treats the call form as meaningful as a statement.
- ✅ **Property-target assignments (`& obj.field = ...`)** [Slice 6] — added a `StoreIndex` runtime control command that pops value+key+base off the eval stack and mutates `base[key] = value` in place on the underlying `ObjectValue.value` Map (Maps are shared by reference, so the change propagates through the variable). New `StorePropertyAssignment(base, key, value)` ParsedHierarchy class emits `EvalStart; base; key; value; StoreIndex; EvalEnd`. Both `lowerExplicitStatement` (for `& obj.field = ...`) and the lowerer-level sibling-pair detection in `lowerStatements` (for bare `obj.field = ...` inside function bodies) detect multi-segment LHS and route to a shared `lowerPropertyTargetAssignment` helper. The base is always built as an `IndexExpression` chain (`obj.theme.bg` → base = `IndexExpression(Var(obj), "theme")`, key = `"bg"`) — committing to value-traversal semantics rather than ink's namespace lookup. Compound forms (`obj.field += 1`) desugar to read-via-IndexExpression + binop + StoreIndex. Arbitrary mixed access-path shapes work — dotted chains, bracket indexers, and mixes thereof (`this.is["a"].access.path = "ok"`).
- **Multiple assignment (`& x, y = 1, 2`)** [Slice 6] — luau supports it; ink doesn't natively.
- **Typed declarations (`local x: number = 1`)** [Slice 6] — type annotation is in the tree (`LuauTypeAnnotationOperation`) but currently ignored.
- ✅ **`const` → real `ConstantDeclaration`** [Slice 6] — `lowerVariableDefinition` now routes `const x = expr` to `ConstantDeclaration` (when a non-null expression is present). The runtime registers the value in `story.constants`, resolves references at compile time, and reassignment trips inkjs's name-conflict check. Side effect: removed the spurious-diagnostic filter for "A variable must be initialized to a number, string, boolean, constant" — that error fired only when `const` lowered to a global var; it no longer applies.
- ⚠️ **`LuauReturnStatement` outside a function** [Slice 10.5] — defensive runtime-export check added to `FlowBase.GenerateRuntimeObject` for `flowLevel === FlowLevel.Story`: scans the Story's `_rootWeave` for any `ReturnType` direct child and emits `"Return statements can only be used inside a function body — found one at file scope."` This catches the bug if it ever happens. In practice the diagnostic doesn't fire today because sparkdown's textmate grammar doesn't currently parse a bare `return X` at top-level into a `LuauReturnStatement` node — it falls through to display-text matching instead. The defensive check is a backstop for a future grammar change that exposes the form. The intra-knot/stitch case is still covered by the existing `flowLevel === FlowLevel.Knot/Stitch` branch.

## Flow-control gaps

- **`LuauTernaryExpression`** [Slice 7] — the `if x then y else z` expression form (e.g. inside `return (...)`). Distinct from the statement form. Lowers to `MultipleConditionExpression`.
- ✅ **Multi-target tunnels (`-> Foo -> Bar`)** [Slice 3] — `buildDivert` walks the chain of nested `Tunnel` grammar nodes, emitting one `Divert` per target. Each non-final target gets `isTunnel=true` (the runtime pushes a return frame); the final target is a plain divert if there's no trailing `->`, a tunnel if there is. Two trailing arrows (`-> X -> Y ->->`) additionally emit a `TunnelOnwards` so the outer caller's frame pops too. Matches inkjs's `InkParser` multi-divert behavior. Validated by `tests/runtime/Diverts.test.ts > Diverts — multi-target tunnels`.
- ✅ **Tunnel-onwards return (`->->`)** — explicit return marker emits a `TunnelOnwards` runtime control command via `buildDivert` / `lowerDivert`. Bare `->->` (pop the tunnel frame) and parameterized `->-> target(args)` (pop then divert) both work. Validated by `Diverts.test.ts > tunnel-onwards-*` (`tunnel-onwards-after-tunnel`, `tunnel-onwards-divert-after-with-arg`, `tunnel-onwards-divert-override`, `tunnel-onwards-with-param-default-choice`) and `Builtins.test.ts > read-count-*` fixtures.
- ✅ **Function-call diverts with arguments (`-> Foo(a, b)`)** [Slice 3] — `buildDivert` lowers each tunnel/divert target's `Parenthetical` arg list into expressions via `lowerExpressionFromContainer` and attaches them to the runtime `Divert`. Parameter binding then happens in `FlowBase.GenerateArgumentVariableAssignments` when the divert's target FlowBase generates its container. Works for direct diverts (`-> place(5)`), tunnel calls (`-> x(1) ->`), tunnel-onwards-with-divert (`->-> b(5+3)`), thread spawns (`<- thread1("red")`), and chained multi-target tunnels (`-> one(1) -> two(2) ->`). Validated by `Diverts.test.ts > divert targets with parameters`, `Diverts.test.ts > complex tunnels (chained parameterized tunnel calls)`, `Diverts.test.ts > tunnel onwards divert after with arg`, `Builtins.test.ts > read-count-variable-target`, and `Multiflow.test.ts > multi-flow save/load with threads`.
- ✅ **`ChoiceConditional` (`* if cond text` guard on a choice)** — sparkdown uses `* if expr ...` (post-mark `if` keyword) rather than ink's `{cond}` prefix. The grammar's choice rule captures `if expr` as a guard expression and the choice lowerer wires it to the runtime's `ConditionalSingleBranch` filter. Validated by `Choices.test.ts > \`{cond}\` guards filter which choices appear`(with`_ if true one`/`_ if false not displayed`fixture) and`Choices.test.ts > once only choices can link back to self via labeled visit count` (`\* if firstOpt [Second choice] -> opts`).
- ✅ **Choice text content** [Slice 3] — `lowerChoice` now walks the parsed `ChoiceWithSuppressedText` / `ChoiceWithNoSuppressedText` captures and populates the three Choice ContentLists per ink's anatomy: `c1` → `startContent` (shared between label and chosen output), `c3` → `choiceOnlyContent` (label only — the bracketed `[...]` text), `c5` → `innerContent` (chosen output only — text after `]`). For the unbracketed form, all text goes into `startContent`. A trailing `Text("\n")` is appended to `innerContent` so the chosen output ends with a newline like a regular display line. `hasWeaveStyleInlineBrackets` is set when the bracket form is used. Unblocks ~30 inkjs Choices/Weaves/Strings/Newlines tests that assert on `currentChoices[i].text` or on `Continue()` after `ChooseChoiceIndex`.
- ✅ **Alternator catch-all (`other`)** [Slice 8] — `match` now treats `other` as a true `isElse=true` catch-all branch (no equality check, runs whenever no earlier named arm matched). `plural` keeps `other` as a string-equality match against `plural.category(n)`'s return value, since CLDR uses "other" as the literal catch-all category name. Implemented in `lowerSparkdownConditionalAlternatorBlock` via a `keyword === "match" && arm.key === "other"` guard.
- ✅ **`plural.category` + `lang.current` runtime support** — `plural(n)|one=...|other=...` desugars to `Conditional(init=plural.category(n))`. `plural.category` is implemented as a new `ControlCommand.CommandType.PluralCategory` in `engine/Story.ts`: pops `n`, reads `lang.current` (default `"en"`) from the variable state, and pushes a `StringValue` holding the CLDR plural category. Rule predicates for en/fr/es/de/it/pt/ru/ja/zh/ko/vi/th/ar/cy live in `engine/PluralRules.ts`; unknown languages fall back to English. Authors can store the language object as `store lang = { current = "en" }` and switch with `lang.current = "fr"` at runtime — the next `plural.category(n)` call observes the change. Authors can also call `plural.category(n)` directly as a stdlib function (validated by `tests/runtime/Sequences.test.ts > plural.category(n) is directly callable`). The grammar's `(?=[(])` lookahead on the conditional alternator rules keeps `plural.X` access-path forms from being absorbed as alternator openings — see `LuauSparkdownConditionalAlternatorBlock` and friends in `sparkdown.language-grammar.yaml`. Inline forms of the conditional alternator (inline `{}` and inline-glued) set `ConditionalSingleBranch.isInline = true` so the runtime doesn't insert leading newlines into each branch's body.
- **Pure-luau alternators** (`LuauConditionalAlternatorBlock`, `LuauSequentialAlternatorBlock` — without `Sparkdown` prefix) [Slice 8] — used inside `return (...)` expression contexts. Need lowerers paralleling the Sparkdown variants.
- **Diverts inside alternator arms** — `lowerArmContent` (in `compiler/lower/utils/alternatorArms.ts`) accumulates every child of an alternator arm as raw text into a single `Text` ParsedObject. A literal `-> target` inside an arm — e.g. `{chain | -> some_label | other text end}` — gets stored as `Text("-> some_label")` rather than recognised as a `Divert`, so the path lookup never runs and control transfer never happens. Fix would route `LuauDivert` (and related) children through the dispatch lowerer instead of dumping them into the text buffer. Blocks porting the full inkjs `divert_to_weave_points` fixture verbatim (alternator-with-divert is the "alternator arm pivots to another path" idiom). Workaround today: hoist the divert out of the alternator into surrounding `if then -> X end` flow.
- ✅ **`->` as a divert-target type annotation** — added `LuauDivertTargetType` grammar rule (matching the literal `->`) and included it in `LuauTypeLiteral`'s `begin` lookahead + pattern list. Authors can write `function cut_to(escape: ->) ... end` (sparkdown's analogue of ink's typed-divert parameter `function f(-> target)`). The annotation is purely declarative — sparkdown's runtime accepts any value as a divert-target if used as one (validated by `Diverts.test.ts > tunnel-onwards-variable-target`); the type marker signals author intent and gives a future typechecker something to validate. Validated by `smoke.test.ts > \`->\` is accepted as a divert-target type annotation`.
- ✅ **Auto-emit `-> DONE` for non-function `scene` / `branch` ends** — `SparkdownCompiler.parseIncrementally` now walks every collected `topLevelFlowBaseObjs` (recursively through `subFlowsByName` to catch nested branches) and appends `Divert(Identifier.Done())` to any non-function flow whose `_rootWeave` doesn't already end with a `Divert` / `TunnelOnwards` / `ReturnType`. Authors get to skip the trailing `-> DONE` boilerplate on most scenes — running off the end of a `scene main` body now terminates cleanly rather than hitting the runtime's "ran out of content" diagnostic. Mirrors the existing `isRootStory` branch in `FlowBase.SplitWeaveAndSubFlowContent`. Validated by `smoke.test.ts > scenes / branches without explicit -> DONE auto-terminate cleanly`.
- ✅ **EXTERNAL bindings (`external message(x)`)** — sparkdown uses the lowercase single-line form (only legal at file scope; not allowed inline behind `&` or inside scenes/branches). The new `LuauExternalDeclaration` grammar rule captures the name + parameter list (reusing `LuauFunctionDeclarationName` / `LuauFunctionParameters`); `lowerLuauExternalDeclaration` produces an inkjs `ExternalDeclaration` parsed object that `Story.AddExternal` registers in `Story.externals`. Call sites (`& message("hi")`, `{multiply(5,3)}`) go through the regular `FunctionCall` → `Divert` lowering — `Divert.GenerateRuntimeObject` already checks `context.IsExternal(target)` and flips `runtimeDivert.isExternal = true`. The runtime API (`BindExternalFunction`, `UnbindExternalFunction`, `EvaluateFunction`, `ObserveVariable`, `ValidateExternalBindings`, lookahead-safety flag) is inherited from inkjs. Validated by `tests/runtime/Bindings.test.ts` (5 of 6 upstream tests; the knot-function-with-narrative shape is design-gated).

## Display-layer gaps

- **Character parenthetical / position** [Slice 4] — `DialogueCharacterParenthetical` (`N (whisper):`) and `DialogueCharacterPosition` (`N [LEFT]:`) aren't captured into tags.
- **Text styling** [Slice 4] — bold `**`, italic `*`, raw backtick, underline `__`, centered `^^`, emphasis tilde / colon — currently pass through as literal punctuation in the Text run.
- ✅ **Escape sequences (`\X`)** [Slice 4] — `processDisplayText` in `lowerDisplay.ts` handles `\<space>` / `\<tab>` / `\<newline>` (paragraph break with trailing space) and `\<other>` (literal) for parity with `InkParser.ContentTextAllowingEscapeChar`.
- ✅ **Trailing ` >` break** [Slice 4] — `processDisplayText` strips the `>` and signals the caller, which appends a separate `Text("\n")` matching InkParser's `terminateWithNewline` path.
- ✅ **Mid-content newlines (block bodies)** [Slice 4] — `\n` becomes `\n ` (newline + trailing space) so a following `{expr}` chunk isn't read as part of an escape sequence.
- **`TextCommand` (`<style>...</style>` inline tags)** [Slice 4] — not handled.
- ✅ **Inline `{expr}` interpolation inside display text** [Slice 4] — `processDisplayBody` in `lowerDisplay.ts` walks the body range, splicing top-level `LuauInterpolatedStringExpression` nodes between text segments. Each lowered `Expression` is marked `outputWhenComplete = true` so the runtime emits its value into the weave content alongside the surrounding `Text` runs. Nested interpolations inside a backtick string in the body are intentionally left to the backtick string's own interpolation machinery.
- ✅ **Inline mid-line divert (`text -> target`)** [Slice D] — `collectTopLevelInjections` in `lowerDisplay.ts` now picks up `Divert` nodes nested inside the display body alongside the existing interpolation handling. When a divert is found, the injection's `from` is anchored at the inner `DivertMark` (rather than the outer `Divert` node) so the whitespace the grammar captured before `->` stays in the text segment — that space is what separates the preceding text from the diverted-to content visually. The lowered `Divert` is spliced into the weave inline; flow transfers to the target scene right after the preceding text is emitted.

## Define-layer gaps

- ✅ **CLASS defines (methods + inheritance + `new`)** — `define` blocks now split into two flavors in `lowerLuauDefine`, keyed on whether the block (or anything it transitively inherits from) declares a METHOD member:
  - **Data defines** (properties only — `define companion as character with ...`) keep the legacy struct path unchanged: compile-time `StructDefinition` registry + flat dot-access property globals, never initialized or serialized at runtime.
  - **Class defines** lower to a GLOBAL DECLARATION whose initializer builds the class table at story start: data properties become class-table defaults, each method becomes a hoisted anonymous `Function` knot with an implicit `self` first parameter stored as a closure value under the method's name, and `as Parent` chains via `setmetatable(Class, { __index = Parent })` (table-form dispatch — plain map walks, no function-form metamethod re-entry on the method-call hot path). Initialization is document order, so parents must be defined before subclasses. `new ClassName(args)` (grammar rule `LuauNewExpression`) lowers to the hidden stdlib constructor `__new(Class, ...args)`: fresh instance table with `__index` → class, `store`-marked property defaults copied INTO the instance (instance-owned from birth → always serialized with the instance; non-store properties read through the chain until written), then the nearest `init` method up the chain is invoked with the instance + args. Receiver dispatch (`penguin:fly()`) rides the existing colon-call value dispatch + `indexThroughMetatable` chain walking. Class-ness is computed document-wide (`CompilationAnnotator.computeClassDefineNames`) since chunks lower with fresh contexts. _Known gap:_ a method-less ROOT class can't be auto-detected — give it a method (or chain it from one) to opt in. Validated by `tests/runtime/DefineClasses.test.ts`.
- ⚠️ **Property modifiers** — `store` now drives `__new`'s copy-into-instance behavior on class defines; `read` / `write` are recorded into hidden `__readProps` / `__writeProps` class lists but NOT yet enforced at runtime. On data defines all modifiers remain dropped (legacy behavior).
- **Nested struct properties** [Slice 9] — `StructPropertyDefinition.level` is hardcoded to `0`. Indented sub-properties for nested structures won't work.
- **Context population for newly-defined structs** [Slice 9] — currently skipped. Existing `populateDefinedStructs` is called only by the InkParser fallback. The new flow needs to call it from the post-compile pipeline once `runtimeStory.structDefinitions` is built. Affects autocomplete and validation for user-defined struct types.
- **Class instances vs save files** — instances are ordinary tables, so the known serialization gaps apply: aliasing and cycles don't survive `JsonSerialisation` (each reference serializes a copy; cycles recurse), and metatables round-trip but `__index` chains serialize the WHOLE CLASS TABLE inline with every instance. Reference-ID serialization is the planned fix and should land before class-heavy content ships.

## Function-layer gaps

- **Anonymous function expressions (`return function(...) ... end`, currying)** [Slice 10] — ink has no first-class functions. Currently produces empty content if encountered in expression context. May need to either translate to a runtime closure mechanism or restrict the language.
- **`local function name()`** [Slice 10] — scoped function declaration. Grammar captures the `LuauScopeModifier`; lowerer ignores it.
- **Return type annotations** and **generics** [Slice 10] — preserved in tree, ignored at the runtime layer. Acceptable for typecheck-only use.
- **Function-end accumulation boundary** [Slice 10] — `end` keywords don't explicitly close scopes at the compiler level. Works for sequential function declarations but fragile for unusual nesting (e.g. function inside a scene body). Tied to the `LuauEndKeyword` critical item (now resolved at the dispatcher; deeper scope-termination work still pending).

## Meta / cross-cutting

- ✅ **`DebugMetadata` on lowered `ParsedObject`s** [all slices] — _dispatcher-level stamping done; per-sub-object refinement deferred._ `LowerContext` exposes `lineNumber(pos)` and `characterNumber(pos)` accessors that return chunk-relative 0-based positions. `lower.ts` wraps the switch dispatch in a wrapper that calls `stampDebugMetadata(block.content, nodeRef.from, nodeRef.to, ctx)` after lowering — every top-level returned ParsedObject gets a `DebugMetadata` derived from its originating node. Nested children without their own metadata inherit via `ParsedObject.get debugMetadata()`'s parent-chain walk. The existing `SparkdownCompiler.remapContent` / `offsetDebugMetadata` pipeline picks these up unchanged: on first encounter it saves the chunk-relative values into `sourceStartLineNumber` / `sourceEndLineNumber` and rebases `startLineNumber` / `endLineNumber` to absolute lines by adding the chunk's start-line offset. Two contexts: the `CompilationAnnotator` builds its context from the codemirror `Text` (sharing the document's line index for performance), while the snapshot tools use `createLowerContextFromSource(source)` which pre-computes line starts from a raw string. Per-sub-object metadata (e.g. attaching a separate `DebugMetadata` to the inner `Identifier` of a `VariableAssignment`) is left for follow-up — diagnostics today get the surrounding statement's range, not the precise sub-token range.
- **`Argument.isByReference` / `isDivertTarget`** [Slice 2] — always `false` because sparkdown grammar has no `ref` / `->` markers. Acceptable as a permanent limitation unless the language is extended.
- **Missing declaration name** (malformed `scene` / `branch` / `function` / `define`) [Slices 2, 9, 10] — currently returns `{}` silently. A validator should surface this as a diagnostic.
